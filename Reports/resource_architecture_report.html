Architectural Stages

Baseline

Commit 5e321bd. Shared splat buffer, single mutable order buffer, full-scene CPU/unsorted order capacity.

1. Frame Resources

Accepted. Per-frame resource ring for uniforms and order buffers, plus in-flight semaphore and mode-local state reset.

2. Private Scene Buffer

Accepted. Static splats upload through staging into a private Metal buffer; packed CPU copy is not retained.

3. Active Order Buffers

Accepted final. CPU/unsorted order buffers sized to active draw count; GPU sort remains power-of-two; CPU/unsorted writers clear unused tails.

4. Single Sort Encoder

Rejected. Reduced encoder churn, but regressed the 100k GPU-sort case enough to revert.

Decision Notes

The order-buffer race was the highest-risk issue, so the accepted design now has an in-flight resource ring and a semaphore. Mode switches clear per-frame mode state before allocating the new mode resources, which avoids state contamination between unsorted, GPU, and CPU paths.

The private splat buffer is architecturally cleaner but increases renderer load time because upload now includes a staging copy and blit. The packed CPU array is local to load and released after upload. CPU sorting still keeps the source splat array because the UI explicitly allows switching to CPU mode after load.

Some frame-time deltas are within short-run variance, so accepted stages were judged on correctness, memory lifetime, and whether they avoided broad regressions. The single-encoder bitonic experiment was measured and rejected.
